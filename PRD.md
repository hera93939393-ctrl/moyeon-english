# 모연 English — PRD

> Product Requirements Document · Main Quest 1 (바이브코딩 웹 서비스)
> 뼈대: **무엇 / 기능 / 동작 / 규칙**

---

## 1. 무엇 (What) — 무엇을, 왜, 누구를 위해 만드는가

**한 줄 소개**
**모연 English** 학원의 정보를 챗봇으로 안내하고, 레벨 테스트 예약을 받아 원장에게 문자로 알려주는 웹 서비스.

**문제 정의**
학부모는 커리큘럼·시간표·셔틀버스·수강료·레벨이 궁금할 때마다 학원에 전화해야 하고,
레벨 테스트 예약도 통화로 일정을 맞춰야 해서 번거롭다. 학원도 같은 문의에 반복 응대하고,
예약 인원이 몰릴 때 정원 관리를 수기로 해야 하는 부담이 있다.

**누가 쓰는가 (사용자)**

| 사용자 | 누구인가 | 왜 쓰는가 |
|---|---|---|
| 학부모 | 자녀를 학원에 보낼지 알아보는 보호자 | 전화 없이 정보 확인, 레벨 테스트 예약·변경·취소 |
| 원장(운영자) | 학원을 운영하는 사람 | 새 예약 문자 알림 받기, 관리자 화면에서 예약·정원 확인 |

## 2. 기능 (Features) — 어떤 기능이 있는가

**핵심 기능 (MVP, 구현 완료)**
1. **정보 안내 챗봇** — 커리큘럼·레벨·시간표·셔틀버스·수강료·초등/중등 프로그램·할인 이벤트 안내 (버튼 + 자유 질문)
2. **오시는 길** — 주소·교통·주차 안내 + 카카오맵/구글지도 링크
3. **레벨 테스트 예약** — 달력에서 날짜 선택 → 시간대별 예약 (정원: 하루 총 30명, 시간대별 2명)
4. **예약 변경·취소** — 예약자 본인이 관리 토큰으로 자신의 예약을 수정·취소
5. **개인정보 보호 규칙** — 수집·이용 동의 체크 필수, DB에 저장된 예약은 **30일 후 자동 삭제**
6. **원장 문자 알림 (SOLAPI 연동)** — 새 예약이 접수되면 원장 휴대폰으로 즉시 SMS 발송
7. **관리자 화면** — 정원 현황(확정 인원/정원), 예약 목록, 리마인더 대상자 수 확인. **로그인 보호 적용**

**남은 기능 (선택 확장)**
- 예약 하루 전 **학부모에게** 리마인더 문자 발송 (현재 `/api/reminders/due`가 대상자 목록만 반환 — 실제 발송 로직은 미구현)
- Claude API 연결 → 자연어 대화형 챗봇으로 고도화

## 3. 동작 (Behavior) — 어떻게 움직이는가

**화면 구성 (3개)**
- 학부모 화면 `/` : 챗봇 + 할인 배너 + 오시는 길 + 레벨 테스트 예약(달력형)
- 관리자 화면 `/admin.html` : 예약 목록·정원 현황 (**로그인 필요**)
- (관리자 로그인은 브라우저 기본 인증창으로 처리)

**사용 흐름**
- 학부모: 접속 → 정보 확인/챗봇 질문 → 달력에서 날짜·시간 선택 → 학생 정보 입력 → 개인정보 동의 → 예약 확정 → 확인 화면(장소·준비물·안내) 표시 → 필요 시 나중에 변경/취소
- 원장: 예약 접수 즉시 **문자 알림 수신** → 관리자 화면 로그인 → 예약 목록·정원 확인

**데이터 구조 (reservations 테이블 — SQLite)**

| 필드 | 타입 | 설명 |
|---|---|---|
| id | 정수(자동증가) | 예약 번호 |
| student_name | 문자열 | 학생 이름 |
| phone | 문자열 | 보호자 연락처 |
| student_grade | 문자열 | 학년 (초1~중3) |
| subject | 문자열 | 학교 이름 |
| slot_id | 문자열 | 예약 날짜\|시간 (정원 계산 기준) |
| note | 문자열 | 추가 상담 내용 |
| status | 문자열 | confirmed / cancelled |
| manage_token | 문자열(UUID) | 본인 예약 변경·취소용 토큰 |
| privacy_agreed_at | 시각 | 개인정보 동의 시각 |
| reminder_status | 문자열 | 리마인더 발송 상태 |
| created_at | 시각 | 접수 시각 (30일 후 자동 삭제 기준) |

*학원 정보(커리큘럼·수강료·오시는 길·할인 등)는 `data/academy.js` 설정 파일로 관리.*

**서버 API**

| 방식 | 주소 | 하는 일 |
|---|---|---|
| GET | `/api/academy` | 학원 공개 정보 조회 |
| POST | `/api/chat` | 챗봇 대화 (키워드 매칭) |
| GET | `/api/reservations/slots` | 예약 가능 시간대 조회 |
| GET | `/api/reservations/status` | 정원 현황 조회 |
| POST | `/api/reservations` | 예약 생성 (Create) + 원장 문자 알림 |
| PATCH | `/api/reservations/:id` | 예약 수정 (Update, 관리 토큰 필요) |
| DELETE | `/api/reservations/:id` | 예약 취소 (관리 토큰 필요) |
| GET | `/api/reservations` | 전체 예약 조회 (Read) — **관리자 전용** |
| GET | `/api/reminders/due` | 내일 리마인더 대상 조회 — **관리자 전용** |

## 4. 규칙 (Rules) — 지켜야 할 규칙 (완성도 · 가드레일)

- **챗봇은 아는 것만 답한다** — 학원 정보에 없는 내용은 구체적으로 물어봐 달라고 안내
- **필수값 검증** — 학생 이름·연락처·학년·학교·시간대·개인정보 동의 중 하나라도 없으면 예약 불가
- **정원 초과 방지** — 시간대별 2명, 전체 30명 초과 시 예약 차단
- **연락처 중복 방지** — 동일 번호로 진행 중인 예약은 1건만 유지
- **개인정보 최소 보관** — 30일 경과 시 DB에서 자동 삭제
- **관리자 화면 보호** — `/admin.html`과 예약 조회 API는 로그인(기본 인증) 없이는 접근 불가
- **에러 대응** — 실패 시 사용자에게 구체적인 이유(정원 마감, 잘못된 입력 등)를 안내

**기술 스택**
- 프론트엔드: HTML / CSS / JavaScript (바닐라)
- 백엔드: Node.js + Express
- DB: SQLite (`node:sqlite`) — 배포 시 유지 또는 클라우드 DB(Neon 등) 검토
- 문자 알림: 솔라피(SOLAPI) SMS API
- 배포: 예정 (Render 등 무료 호스팅 검토)
