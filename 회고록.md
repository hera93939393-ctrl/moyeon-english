# 회고록 — 모연 English를 만들며

> Main Quest 1: 바이브코딩으로 웹 서비스 만들기

## 무엇을 만들었나

학원 안내 챗봇과 레벨테스트 예약 시스템 "모연 English"를 만들었습니다. PRD 작성부터
시작해서, 화면 만들기 → 예약 저장과 정원 관리 → 문자 알림 → 배포까지 5단계를 거쳤습니다.
처음엔 Claude와 진행했고, 크레딧 이슈로 유료결재중인 ChatGPT(Codex)로 기획·화면·기본 백엔드까지 진행했습니다.
이후 Claude와 함께 예약 저장(DB), 문자 알림, 관리자 보호, 배포까지 마무리했습니다.

➤ 시작할땐 머리가 멍하고 내가 할수 있을까 두려움이 들었지만, 10여일간 꾸준히 배워온 내용을 회고하며 AI와 함께 차근차근 단계를 밟아 따라가다 보니 신기하게 문제점들이 해결되고 결과물이 나와 기쁘고 뿌듯했습니다.

## AI와 협업하며 배운 점

**1. "무료"·"간단함"이라는 말을 그대로 믿으면 안 된다는 것을 배웠습니다.**
문자 알림 기능 하나를 붙이는 데만 세 번을 갈아엎었습니다. 네이버 클라우드는 개인 계정으로는
아예 막혀 있었고(사업자 등록 필요), 알리고는 가능했지만 최소 충전 금액이 5만원이라 부담스러웠습니다.
결국 솔라피(SOLAPI)를 찾아 계좌이체 1,000원부터 시작할 수 있다는 걸 확인하고 나서야 진행할 수
있었습니다. AI가 안내해준 정보("무료 크레딧을 준다")도 실제로는 결제 화면이 먼저 뜨는 등
다르게 나타나는 경우가 있어서, **AI의 설명도 실제로 눌러보고 확인하는 과정이 꼭 필요하다**는
걸 몸으로 배웠습니다.  (Claude는 3번 유로서비스 추천 후 무료서비스 더는 못찾겠다 했지만, ChatGPT는 한번에 찾아줌)

**2. 나중에 문제가 될 걸 미리 점검하는 습관이 생겼습니다.**
배포 직전에 "무료 서버가 잠들었다 깨어나면 예약 데이터가 사라질 수 있다"는 걸 미리 확인하고,
로컬 SQLite에서 클라우드 DB(Neon)로 옮긴 뒤에 배포를 진행했습니다. 만약 이걸 모르고 그냥
배포했다면, 채점 중에 예약 데이터가 사라지는 낭패를 볼 뻔했습니다.

**3. 기능과 비용의 균형을 스스로 결정하는 경험을 했습니다.**
업로드한 커리큘럼 파일을 AI가 읽고 자연스럽게 답하게 만들 수도 있었지만, Claude API 사용에
결제가 필요하다는 걸 알고 나서 "지금은 무료인 파일 검색 방식으로 충분하다"고 판단해 방향을
바꿨습니다. 이런 선택을 스스로 내려본 것이 좋은 경험이었습니다. 다만 AI 연동 코드 자체는
삭제하지 않고 남겨뒀습니다 — `.env`에 API 키만 넣으면 언제든 AI 답변 방식으로 전환할 수
있도록, "구현은 하되 기본은 무료 방식으로 동작"하게 설계했습니다.

➤ 2개의 AI툴을 활용해 보니 무조건 Claude가 정보서치와 코드를 잘짠다고 볼수 없었어요. 문자서비스 이용시 무료로 쉽게 보낼수 있는 방법을 
Claude는 못찾은 반면 ChatGPT는 한번에 찾았고, Claud에게 ChatGPT의 코딩실력을 비판적으로 분석해보라고 했더니 굉장히 잘 만들었다는 평가를
받았습니다. 부분별로 강점이 있는게 AI툴이구나 생각했습니다

## 어려웠던 점

- SMS 연동에서 서비스를 세 번이나 바꾸면서, 매번 새로운 가입·인증 절차를 겪어야 했던 점
- Render 배포 중 갑자기 카드 등록을 요구받아 당황했던 점 (결국 카드가 필요 없는 Railway로 옮김)
- git 저장소가 이상한 폴더(`.git-data`)에 저장되어 있어서, GitHub에 올리기 전에 권한 문제부터
  해결해야 했던 점 — 코딩 외적으로도 신경 쓸 게 많다는 걸 느꼈습니다

➤ 내가 혼자서 작업을 했다면 SMS나 새로 연결해야 할 프로그램을 찾을때 시간이 많아 걸려 힘들었을텐데(비교 등) 비교적 빠른 시간내 AI의 추천을 받아 고르고 연결시켜 완성시킬수 있어서 AI 덕분에 훨~씬 수월하게 작업한것 같습니다. 물론 코딩은 말할것도 없구요!
